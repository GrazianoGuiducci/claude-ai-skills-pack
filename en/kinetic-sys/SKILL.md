4. **Approval Gate**: Present the simulation to the user. No physical action without explicit confirmation.

**Phase B -- Actuator Orchestration**
1. **Terminal Selection**: Which device/actuator/system executes the action?

3. **Output Format**: Not descriptive text. Structured control instructions (JSON, API calls, commands).

**Phase C -- Sensory Reconciliation**
1. **Loop Closure**: After the action, read the sensors. Actual final state.
2. **Comparison**: Final state vs original intent.
3. **Correction**: If discrepancy -> calculate delta and correct (if possible) or report.
